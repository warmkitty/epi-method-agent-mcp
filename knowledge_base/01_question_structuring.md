# Skill 1：研究问题结构化（Question Structuring）

## 输入要求

执行本 Skill 前，确认已获取以下信息（缺失时向用户询问）：
- 疾病/适应症
- 药物/干预措施
- 客户需求（洞察/论文/医保准入/HEOR/BIA）
- 可用数据源（医保/EMR/Registry/Claims）
- 目标用途

## 执行步骤

### Step 1：判断研究类型

根据客户需求和输入信息，将研究归类为以下类型之一：

| 研究类型 | 特征 | 典型问题 |
|---------|------|---------|
| 描述性 | 无对照组，描述分布/趋势 | 患病率、发病率、治疗模式、医疗资源利用 |
| 比较有效性研究(CER) | 有干预组+对照组 | 药物A vs 药物B 的有效性/安全性 |
| 安全性研究 | 关注不良事件/风险 | 药物A是否增加XX风险 |
| 经济负担 | 成本/资源消耗 | 疾病X的总经济负担、药物A vs B的成本差异 |
| BIA（预算影响分析） | 医保支付视角 | 新药纳入医保的预算影响 |
| 预测模型 | 预测个体风险/结局 | 预测XX事件的发生风险 |

### Step 2：构建 PICO/PECO 框架

**干预性研究（有治疗/干预措施）使用 PICO：**
- **P (Population)**：目标人群，包含诊断和排除标准
- **I (Intervention)**：待评估的干预/治疗药物
- **C (Comparator)**：对照（常规治疗/安慰剂/同类药物/无治疗）
- **O (Outcome)**：主要结局 + 次要结局
- **T (Time)**：随访时间（可选）

**观察性/暴露研究使用 PECO：**
- **P (Population)**：目标人群
- **E (Exposure)**：暴露因素（如某种药物、某种行为）
- **C (Comparator)**：对照暴露
- **O (Outcome)**：结局
- **T (Time)**：随访时间

**PICO/PECO 构建注意事项：**
1. 人群定义必须可操作（有具体的 ICD/药品编码依据）
2. 干预和对照必须明确时点和持续时间
3. 结局必须有明确的测量时间和测量方式
4. 如为回顾性研究，PICO 必须是回顾后可实际定义的

### Step 3：明确主要和次要研究目标

**主要研究目标**：只有一个，直接回答核心问题

**次要研究目标**：补充分析，不超过3-5个

格式：
```
主要研究目标：
1. [具体、可检验的假设或描述性目标]

次要研究目标：
1. ...
2. ...
```

### Step 4：明确"可回答"和"不适合回答"的问题

**可回答问题**：在当前数据和方法条件下可以可靠回答的问题

**不适合回答问题**：数据不支持、偏倚过大、或方法上不可识别的问题
（这部分很重要，体现方法学诚信）

格式：
```
可回答问题：
- ✅ ...

不适合回答问题：
- ❌ ...（原因：...）
```

## 输出模板

```markdown
【研究问题结构化结果】

## 研究类型
[描述性 | 比较有效性 | 安全性 | 经济负担 | BIA | 预测模型]

## PICO/PECO 框架
| 要素 | 定义 |
|------|------|
| P/Population | ... |
| I/Intervention 或 E/Exposure | ... |
| C/Comparator | ... |
| O/Outcome | 主要：... 次要：... |
| T/Time | ... |

## 研究目标
**主要**：...

**次要**：
1. ...
2. ...

## 可回答问题
- ✅ ...

## 不适合回答问题
- ❌ ...（原因：...）

## 备注
[数据限制、方法学注意事项]
```

## 质量检查清单

输出前确认：
- [ ] 研究类型判断有明确依据
- [ ] PICO/PECO 各要素均可操作化定义
- [ ] 主要研究目标只有一个
- [ ] "不适合回答的问题"已诚实列出
- [ ] 术语使用准确（参考 terminology.md）
