# Skill 2：Fit-for-purpose 数据适用性评价

## 核心理念

不是所有研究问题都适合用特定数据源回答。"Fit-for-purpose"评价的目的是在研究设计初期就明确数据的能力边界，避免"数据不支持还硬做"的常见错误。

## 七维评价框架

### 维度 1：人群识别能力

**评估内容**：
- 研究目标人群是否可以通过数据中的诊断编码、实验室结果等准确识别
- 人群定义的灵敏度和特异度如何
- 是否有足够的人群量满足统计要求

**典型问题**：
- 疾病编码是 ICD-10 还是国家卫健委编码？版本一致吗？
- 门诊诊断 vs 住院诊断：哪个用于人群识别？
- 无诊断编码的疾病（如某些罕见病）如何处理？
- 新发 vs 患病：数据能否区分？

**医保数据特点**：通过"诊断+药品+检验"组合可提高灵敏度，但特异度可能降低

### 维度 2：暴露/治疗识别能力

**评估内容**：
- 研究药物/干预措施能否被准确识别
- 处方信息是否包含剂量、频次、疗程
- OTC 药物和非处方渠道的药物能否捕捉

**典型问题**：
- 药品编码是国码/省码还是医院码？能否映射到标准编码？
- 处方数据 vs 实际使用：数据反映的是处方还是实际用药？
- 药品是否在医保目录内？（目录外药物无法捕捉）
- 给药途径是否能识别？

**医保数据特点**：医保结算数据只有医保目录内药品，且只能捕捉到结算记录，无法确认实际服药依从性

### 维度 3：结局捕捉能力

**评估内容**：
- 主要结局和次要结局是否能在数据中被可靠测量
- 结局的灵敏度和特异度
- 结局事件的时间精度

**典型问题**：
- 结局是否需要住院才能捕捉？（门诊结局可能遗漏）
- 死亡数据是否包含死因？
- 实验室指标是否有连续测量值？
- 不良事件是否有专门的编码？

**医保数据特点**：住院结局捕捉较好，门诊结局和实验室数据通常不可用

### 维度 4：混杂变量支持能力

**评估内容**：
- 关键混杂因素是否在数据中可获取
- 混杂变量的测量质量和时间精度
- 是否有足够的变量进行倾向评分计算

**必须评估的混杂变量**：
- 人口学：年龄、性别、地区
- 疾病严重程度：合并症、既往住院次数、Charlson/Deyo指数
- 用药历史：基线用药、LOT
- 生活方式：通常不在医保数据中

**医保数据特点**：人口学和疾病信息较好，但生活方式（吸烟、BMI）、实验室指标（HbA1c、eGFR）通常不可用

### 维度 5：随访完整性

**评估内容**：
- 人群是否可以持续追踪
- 脱落/失访的比例和原因
- 右删失机制是什么

**典型问题**：
- 医保数据是否可以跨医院追踪？
- 离开参保地或变更保险后的情况
- 死亡后是否仍有记录？
- 数据的时间跨度是否足够？

**医保数据特点**：通常以参保关系为追踪基础，失保或断保会导致删失，需分析失保原因

### 维度 6：成本/资源利用分析能力

**评估内容**：
- 直接医疗成本是否可准确计算
- 间接成本（误工、交通等）是否可获取
- 成本数据是否包含所有相关类别（药品、检查、住院、手术）

**典型问题**：
- 成本是实际支付金额还是标准定价？
- 自付部分是否能区分？
- DRG/DIP 支付下成本如何归因？
- 药品是否有集采价格信息？

**医保数据特点**：直接医疗成本（医保支付部分）可计算，自付和目录外费用需补充数据

### 维度 7：主要局限

综合以上 6 个维度的评估，总结数据的主要局限。

**必须说明的局限**：
- 影响研究结论可信度的关键局限
- 可能导致偏倚的数据缺陷
- 结果外推性的限制

## 可行性结论（四档）

| 结论 | 含义 | 建议后续行动 |
|------|------|------------|
| **可直接做** | 数据全面支持研究问题 | 进入统计分析设计 |
| **可探索做** | 数据基本支持但有明显局限 | 需增加敏感性分析、谨慎解释结果 |
| **需外部数据补充** | 核心变量缺失，数据无法独立完成 | 考虑 linkage（如EMR+医保）或调整研究问题 |
| **不建议做** | 数据无法支持核心研究问题 | 重新定义研究问题或更换数据源 |

## 数据源能力对比

### 中国医保数据（本地化）

| 能力 | 支持 | 说明 |
|------|------|------|
| 诊断信息 | ✓ | ICD-10 编码，住院诊断较可靠 |
| 药品处方 | ✓ | 医保目录内药品，有编码和日期 |
| 住院信息 | ✓ | 入院/出院日期、科室、费用明细 |
| 门诊处方 | ✓ | 但依从性无法确认 |
| 实验室检查 | ✗~△ | 部分统筹地区有，通常不全 |
| 生活方式 | ✗ | 不可用 |
| 死亡信息 | △ | 部分地区有住院死亡记录 |
| 成本/费用 | ✓ | 医保支付部分 |

### EMR 数据

| 能力 | 支持 | 说明 |
|------|------|------|
| 诊断信息 | ✓ | 门诊+住院，更详细 |
| 药品处方 | ✓ | 含剂量、频次 |
| 住院信息 | ✓ | 详细病程记录 |
| 实验室检查 | ✓ | 连续测量值可用 |
| 影像学 | △ | 结构化程度低 |
| 生活方式 | △ | 病程记录中可能有 |
| 死亡信息 | △ | 仅住院死亡 |
| 成本/费用 | △ | 可能不含医保外费用 |
