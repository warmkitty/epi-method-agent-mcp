# Skill 3：Target Trial Emulation（目标试验模拟）

## 核心理念

Target Trial Emulation（TTE）由 Hernán & Robins 提出，核心思想是：

> **在观察性数据中设计研究时，先构想一个理想 RCT（目标试验），然后尽可能在真实世界数据中模拟这个 RCT 的每个要素。**

如果某个 RCT 要素无法在观察性数据中被模拟，这本身就是一个重要的研究局限。

## TTE 九要素框架

### 1. Eligibility Criteria（入组标准）

**目标试验**：RCT 的入排标准
**观察性模拟**：通过诊断编码、实验室值、用药记录定义

**关键注意事项**：
- 入组标准必须基于 Time Zero 前的信息（不能有预后信息泄露）
- 活动性查找偏倚（active comparator bias）：使用活动性对照组（如"已开始用药的患者"）而非"未用药"对照
- 新用户设计（new user design）：首选新用药者，排除已用药者

**模板**：
```
治疗组入组标准：
- [诊断条件]
- [年龄条件]
- [排除标准1]
- [排除标准2]
- 既往X天未使用过目标药物/同类药物

对照组入组标准：
- [同上人群条件]
- [对照组特定条件]
```

### 2. Treatment Strategies（治疗策略）

**目标试验**：RCT 随机分配的治疗方案
**观察性模拟**：定义什么算"接受治疗"

**关键注意事项**：
- Intention-to-treat (ITT)：从 Time Zero 开始评估，不管后续是否停药/换药
- Per-protocol (PP)：评估"持续用药"的效果，但引入了选择偏倚
- As-treated：按实际用药情况评估，引入 immortal time bias

**模板**：
```
治疗组： initiation of [药物A]（意向治疗原则）
对照组： initiation of [药物B]（意向治疗原则）
```

### 3. Assignment（分组机制）

**目标试验**：随机化
**观察性模拟**：用条件（如"开始使用药物A的第一个处方日期"）替代随机化

**关键注意事项**：
- 明确说明分组机制不是随机的
- 分组时点即 Time Zero
- 分组条件必须在 Time Zero 之前满足

### 4. Time Zero（Time Zero 定义）⚠️ 最重要

**定义**：每个参与者进入研究的时刻，也是随访开始和基线变量测量的时间点。

**绝对原则**：
1. Time Zero 必须对所有参加者定义一致
2. Time Zero 之后的事件不能用于入组标准
3. 治疗组和对照组的 Time Zero 必须使用相同的"时间轴"

**常见错误**：

| 错误模式 | 问题 | 正确做法 |
|---------|------|---------|
| 治疗组用处方日期，对照组用随机日期 | 引入 immortal time bias | 对照组也用一个明确的"事件日期"（如同类药首次处方） |
| 用出院日期作为 Time Zero | 出院前已接受治疗，产生 immortal time | 用入院日期或治疗开始日期 |
| 入组后使用基线信息排除患者 | 选择偏倚 | 排除标准只能基于 Time Zero 前的信息 |
| 对照组入组时间早于治疗组 | 时序偏倚 | 两组使用相同的入组日历时间窗口 |

**推荐做法**：
```
Time Zero 定义：
- 治疗组：首次处方药物A的日期
- 对照组：首次处方药物B的日期
- 两组必须在同一个日历时间窗口内入组（如2018-01-01至2023-12-31）
```

### 5. Follow-up（随访）

**目标试验**：RCT 的随访计划
**观察性模拟**：从 Time Zero 到结局事件/删失的时间

**关键注意事项**：
- 随访起点：Time Zero
- 随访终点：结局事件发生 / 研究截止日期 / 脱落（失保、转保等）
- 治疗组停药不等于随访结束（ITT原则）
- 不允许前瞻性回顾（look-ahead bias）

**模板**：
```
随访起点：Time Zero
随访终点（按优先级）：
1. 结局事件发生
2. 死亡
3. 失保（参保关系终止）
4. 研究截止日期
```

### 6. Outcome（结局）

**目标试验**：RCT 的主要终点和次要终点
**观察性模拟**：通过诊断编码、实验室值等定义结局事件

**关键注意事项**：
- 结局必须在随访期内被测量
- 结局定义需要验证（如：只算住院事件？还是门诊也算？）
- 多次发生的事件 vs 首次事件：需要明确
- 结局可能受治疗影响（如：用药后更频繁就诊 → 结局检出率提高 → 检出偏倚）

### 7. Causal Contrast（因果对比）

**目标试验**：RCT 要回答的因果问题
**观察性模拟**：需要用统计学方法估计的因果效应

**常见因果对比类型**：
- **ATE** (Average Treatment Effect)：整个目标人群的平均因果效应
- **ATT** (Average Treatment Effect on the Treated)：治疗组人群的平均因果效应（观察性研究最常估计的）
- **ACE** (Average Causal Effect)：与 ATE 类似

**在 PICO 中的体现**：
```
因果问题：在目标人群P中，策略A与策略B相比，对结局O的影响是什么？
统计量：HR / RD / OR / 均值差
```

### 8. Analysis Plan（分析方案）

**目标试验**：RCT 的统计分析计划
**观察性模拟**：基于 TTE 框架的分析方案

**必须包含的要素**：
1. 描述性分析（基线特征表）
2. 未调整分析（crude comparison）
3. 调整分析（混杂控制方法）
4. 敏感性分析
5. 亚组分析

### 9. Key Deviations from Ideal RCT（与理想RCT的偏离）

**核心产出**：诚实地列出观察性设计无法模拟的 RCT 要素。

**常见偏离**：
| RCT 要素 | 观察性偏离 | 影响 |
|---------|-----------|------|
| 随机化 | 无随机化，存在混杂 | 需要用统计方法调整 |
| 分组盲法 | 无盲法 | 可能存在处方偏倚 |
| 依从性 | 无法控制 | 可用 ITT/PP 两种分析 |
| 随访完整性 | 可能不完整 | 知情删失偏倚 |
| 结局评估 | 可能不统一 | 结局错分 |
