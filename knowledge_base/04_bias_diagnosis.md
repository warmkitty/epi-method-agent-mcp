# Skill 4：偏倚诊断（Bias Diagnosis）

## 核心理念

偏倚诊断是 Epi-Method Agent 中最重要的 Skill。
**即使其他 Skill 都完美执行，如果偏倚未识别和控制，研究结论可能完全错误。**

偏倚诊断的目的：
1. 系统性地识别研究中可能存在的偏倚
2. 评估每种偏倚对结果方向的影响
3. 提出应对策略并评估残余风险
4. 最终给出因果推断的可信度评级

## 偏倚检查流程

```
收到 TTE 框架 + 数据源特征
        ↓
逐一检查 10 种偏倚
        ↓
识别"存在"的偏倚（非所有偏倚都存在）
        ↓
对每个识别到的偏倚，输出六列表
        ↓
按影响程度排序
        ↓
给出因果推断可信度评级
```

## 10 种偏倚检查清单

### 1. Confounding by Indication（适应症混杂）⭐ 最高频

**发生机制**：
- 处方决策受患者病情严重程度影响
- 病情本身也影响结局
- 导致治疗与结局之间的伪关联

**典型场景**：
- 重症患者更可能接受强效药物 → 即使药物无效，重症组结局更差
- 医生根据临床特征选择药物 → 这些特征同时预测结局

**诊断方法**：
- 比较基线特征表：两组是否有明显差异
- 检查疾病严重程度的代理变量
- 评估处方模式是否合理（如：新药先给轻症患者？还是给重症患者？）

**应对策略**：
- 倾向评分方法（IPTW / Matching / Stratification）
- 多变量回归调整
- 新用户设计 + 活动性对照
- 负对照结局检验（negative control outcomes）
- 工具变量（IV，如适用）
- E-value（量化未测量混杂的阈值）

**残余风险**：中-高（尤其当关键混杂变量不可获取时）

---

### 2. Immortal Time Bias（不朽时间偏倚）⭐⭐ 最危险

**发生机制**：
- 暴露状态在 Time Zero 之后才确定
- 从 Time Zero 到暴露确定之间的"不朽时间"，参与者不会发生结局
- 暴露组拥有额外的"安全时间"，人为降低了事件率

**典型场景**：
- "用药超过6个月的患者" vs "未用药"：前6个月是不朽时间
- "二次住院后使用药物"的患者：第一次住院到第二次住院之间是不朽时间
- "住院期间使用药物"的患者：入院到处方日之间的时间

**诊断方法**：
- 检查暴露定义是否依赖随访期内的事件
- 检查 Time Zero 和暴露确定之间是否有时间差
- 画出研究设计的时间轴，检查是否存在"暴露前的不朽时间"

**应对策略**：
- ✅ 使用 intention-to-treat 设计：从首次处方开始定义暴露
- ✅ 使用 landmark 分析：在固定时间点定义暴露
- ✅ 时间依赖性暴露定义（time-varying exposure）
- ❌ 不要按"是否完成了X个月的用药"来定义暴露组

**残余风险**：低（只要设计正确可完全避免）

---

### 3. Selection Bias（选择偏倚）

**发生机制**：
- 入组过程导致研究人群不代表目标人群
- 失访/删失与暴露和结局都相关

**典型场景**：
- 健康体检者效应（healthy user effect）：主动就医的患者更健康
- 入组条件需要"至少一次门诊就诊"，排除了无症状患者
- 选择性失访：某组更容易退出研究

**诊断方法**：
- 比较入组和未入组人群的特征
- 分析失访/删失的模式
- 检查入组条件是否引入了预后差异

**应对策略**：
- 使用 new user design 减少 healthy user bias
- 使用 active comparator 减少 confounding by indication
- 分析失访与基线特征的关系
- 逆概率删失加权（IPCW）

**残余风险**：中

---

### 4. Misclassification Bias（错分偏倚）

**发生机制**：
- 暴露、结局、或混杂变量的测量存在错误
- 可能是非差异性的（random error）或差异性的（systematic error）

**典型场景**：
- ICD 编码错误：如将高血压误编为正常
- 药品编码不准确：如将规格或剂型编错
- 结局定义不准确：如将"疑似"和"确诊"混为一谈
- BMI/实验室值缺失导致用药指征判断错误

**诊断方法**：
- 用编码验证研究评估编码准确率
- 比较不同来源的诊断信息（如门诊 vs 住院）
- 分析编码一致性

**应对策略**：
- 使用多种编码组合定义（如：诊断 + 用药 + 检验）
- 敏感性分析：改变定义标准的宽严程度
- 使用算法验证队列（validation subcohort）

**残余风险**：中-低（取决于编码质量）

---

### 5. Informative Censoring（知情删失）

**发生机制**：
- 删失（如失保、换数据源）不是随机的，而是与暴露和/或结局相关
- 如果删失与结局相关，标准生存分析会产生偏倚

**典型场景**：
- 重症患者更可能转院 → 转院后无法追踪 → 删失 ≠ 随机
- 治疗有效者更可能继续参保 → 生存分析中对照组被不成比例地删失
- 患者因死亡而终止参保 → 参保终止是死亡的前驱

**诊断方法**：
- 分析删失率在两组间是否有差异
- 分析删失与基线特征的关系
- 画 Kaplan-Meier curves for censoring

**应对策略**：
- 逆概率删失加权（IPCW）
- 将删失事件本身作为一个结局进行竞争风险分析
- 敏感性分析：假设删失者全部发生/全部不发生结局

**残余风险**：中（医保数据中参保变动是主要删失来源）

---

### 6. Missing Data（缺失数据）

**发生机制**：
- 关键变量存在缺失值
- 缺失不是完全随机的（MCAR），而是 MAR 或 MNAR

**典型场景**：
- 医保数据中无生活方式信息（吸烟、饮酒、BMI）
- 实验室检查只有部分患者做过
- 合并症诊断不全（特别是轻症患者可能未就诊）

**诊断方法**：
- 评估各变量的缺失率和缺失模式
- 分析缺失是否与暴露/结局相关

**应对策略**：
- 多重插补（Multiple Imputation）
- 完全病例分析（Complete Case Analysis，如有充分理由）
- E-value 评估未测量变量对结论的潜在影响
- 定量偏倚分析（Quantitative Bias Analysis, QBA）

**残余风险**：高（未测量变量无法完全控制）

---

### 7. Calendar-Time Bias（日历时间偏倚）

**发生机制**：
- 入组时间不同，暴露了不同的处方习惯、治疗标准、诊断精度
- 治疗标准随时间变化

**典型场景**：
- 2018年入组 vs 2023年入组的患者，面临的临床指南不同
- 新药刚上市时处方给"最后一线"患者，后来变成"二线"
- 诊断标准更新导致入组人群随时间变化

**诊断方法**：
- 按日历年/季度分层检查基线特征和治疗模式
- 画入组时间与研究变量的关系图

**应对策略**：
- 入组时间窗口限定（如限定2-3年）
- 入组时间作为协变量纳入模型
- 按日历时间分层分析

**残余风险**：低（如正确控制了日历时间）

---

### 8. Competing Risk（竞争风险）

**发生机制**：
- 存在多个互斥的结局事件
- 发生了一个事件后，其他事件不可能发生
- 标准生存分析（如 Kaplan-Meier、Cox）假设删失是独立的，但竞争风险违反这一假设

**典型场景**：
- 研究心血管事件风险，但老年患者高死亡率 → 死亡是竞争事件
- 研究药物导致的肝毒性，但患者先死于癌症 → 癌症死亡是竞争事件
- 评估再入院风险，但部分患者在再入院前死亡

**诊断方法**：
- 检查是否有与主要结局互斥的其他结局
- 估计累积发生率函数（CIF）
- 比较有/无竞争风险调整的结果差异

**应对策略**：
- Fine-Gray 子分布风险模型
- 累积发生率函数（CIF）估计
- 同时报告 cause-specific HR 和 subdistribution HR

**残余风险**：低（如正确使用竞争风险模型）

---

### 9. Depletion of Susceptibles（易感者耗竭）

**发生机制**：
- 随着时间推移，高风险（易感）个体逐渐被"事件"消耗
- 存活者群体的风险特征随时间变化
- 导致处理效应在不同随访时间点不一致（HR 随时间变化）

**典型场景**：
- 手术 vs 药物治疗：手术的短期死亡率较高，但长期受益更大
- HR 随时间变化的趋势：治疗组的早期保护效应消失

**诊断方法**：
- 检查 log(-log(survival)) 曲线：两组是否平行
- Schoenfeld 残差检验：比例风险假设是否满足
- 按随访时间分层分析

**应对策略**：
- 分层 Cox / time-dependent Cox
- 加速失效时间模型（AFT）
- 分别报告短期和长期效应
- 灵活参数模型（如 Royston-Parmar）

**残余风险**：低（如识别并处理了 PH 违反）

---

### 10. Coding Drift / Upcoding（编码漂移/升级）

**发生机制**：
- 编码标准/实践随时间变化
- 医院为获得更高 DRG/DIP 支付而故意升级编码（upcoding）
- 新版 ICD 编码的引入导致分类变化

**典型场景**：
- DRG 支付制度实施后，医院倾向于添加更多合并症诊断（CC/MCC upcoding）
- ICD-10 第二版调整导致某些疾病分类变化
- 电子病历系统的编码提示功能改变了编码行为

**诊断方法**：
- 分析各年份/季度的编码分布变化
- 比较编码变化的"拐点"是否与政策变化一致
- 敏感性分析：排除"疑似 upcoding"的记录

**应对策略**：
- 入组时间限定，避免跨编码标准变化时期
- 敏感性分析：改变编码定义
- 使用编码验证队列评估编码一致性

**残余风险**：中（除非进行专门验证）

---

## 因果推断可信度评级

所有偏倚评估完成后，给出整体评级：

| 评级 | 含义 | 适用场景 |
|------|------|---------|
| **高** | 设计合理，主要偏倚已控制，残余风险低 | RCT质量证据 |
| **中** | 设计基本合理，存在可控偏倚，但关键假设成立 | 充分调整的 TTE + 良好的敏感性分析 |
| **低** | 存在重要偏倚或关键数据缺失，结论需谨慎解释 | 关键混杂变量不可用或偏倚方向不确定 |
| **不可靠** | 设计缺陷导致无法区分因果和混杂 | 如存在未控制的 immortal time bias |
